# Лапа помощи

Веб-приложение для помощи животным: каталог приютов и волонтёров, срочные заявки, личные кабинеты пользователя, волонтёра и организации.

## Требования

- **Node.js** 20+
- **npm** 10+
- Запущенный бэкенд API (по умолчанию `http://127.0.0.1:8000`)

## Переменные окружения

Скопируйте `.env.example` в `.env.local` в корне проекта:

```env
NEXT_PUBLIC_API_URL=http://127.0.0.1:8000
```

## Запуск в режиме разработки

1. Установите зависимости:

```bash
npm install
```

2. Создайте `.env.local` (см. выше).

3. Запустите бэкенд на порту **8000**.

4. Запустите фронтенд:

```bash
npm run dev
```

Сайт: [http://localhost:3000](http://localhost:3000).

## Структура проекта

```
lapa-pomoshi/
├── public/              # Статические файлы (изображения, иконки)
├── src/
│   ├── app/             # Маршруты Next.js (App Router)
│   │   ├── (auth)/      # Вход и регистрация
│   │   ├── (public)/    # Публичные страницы (главная, каталог, срочные, помощь)
│   │   └── (protected)/ # Закрытые разделы (профили, кабинет организации, сообщения)
│   ├── entities/        # Сущности домена (animal, organization, user): модели, API, UI
│   ├── features/        # Пользовательские сценарии (авторизация, анкета усыновления, заявки)
│   ├── widgets/         # Крупные блоки интерфейса (header, footer, списки, сайдбар)
│   ├── shared/          # Общий код: API-клиент, endpoints, UI-кит, утилиты, типы
│   └── proxy.ts         # Проверка авторизации для защищённых маршрутов
├── next.config.ts       # Конфиг Next.js (rewrites на бэкенд)
├── package.json
└── .env.example         # Пример переменных окружения
```

Защищённые маршруты проверяются в `src/proxy.ts` (роль и cookie `auth_token`).
